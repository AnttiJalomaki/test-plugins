# Ntrip and IGS real-time publication

## IGS station publication profile

An IGS real-time station should sustain 1 Hz data with under two seconds of latency. Publish over Ntrip to two broadcasters when possible. Otherwise, use the nearest Global or Regional Broadcaster. Identify whether the stream is native receiver output or generated by external software.

## Ten-character IGS mountpoint names

Station streams use `SSSSMRCCCF`: a four-character station code, marker digit, receiver digit, ISO country code, and format/stream digit. Final digit `0` identifies the most complete RTCM 3 stream; for example, `POTS00DEU0`.

Product streams use `TTTTXXAAAF`: a four-character type, two-digit solution identifier, three-character agency code, and format digit. Format `0` is RTCM 3, `1` is IGS-SSR, and `2`–`9` are reserved. `SSRA` and `SSRC` identify SSR streams whose orbits refer to the antenna phase center and center of mass, respectively. Other defined types are `DCBS`, `IONO`, `TROP`, and `BCEP`.

```text
SSRA00DLR1
^^^^ stream type   ^^ solution   ^^^ agency   ^ format
```

## Broadcaster mirroring and topology

Every IGS Global Broadcaster should expose all official IGS data and product streams under identical mountpoint names. The same name at different broadcasters is assumed to carry the same real-time data. Broadcasters synchronize catalogs and may pull a missing station stream from another caster, but keep the chain short and near the source. A Global Broadcaster should also have a local hot or cold standby.

## Ntrip sourcetable layouts

Sourcetable fields are semicolon-separated and ordered as follows:

```text
CAS;host;port;identifier;operator;nmea;country;latitude;longitude;fallback-host;fallback-port;misc
NET;identifier;operator;authentication;fee;web-net;web-str;web-reg;misc
STR;mountpoint;identifier;format;format-details;carrier;nav-system;network;country;latitude;longitude;nmea;solution;generator;compression;authentication;fee;bitrate;misc
```

For STR records, `carrier` is `0` for none, `1` for L1, or `2` for L1/L2. `nmea=1` requires the client to send an approximate position. `solution` is `0` for a single base or `1` for a network solution. Authentication is `N`, `B`, or `D` for none, Basic, or Digest; fee is `N` or `Y`.

In a CAS record, the NMEA flag instead indicates whether the caster accepts an incoming approximate-position message. Fallback `0.0.0.0;0` means no fallback.

## IGS sourcetable integrity and provenance

Include CAS, NET, and STR records with no empty fields. Give each network named by an STR record a corresponding NET record. Use network `IGS` for accepted station streams and `PROPOSED` for proposed stations. Point the NET `web-str` field to the station RINEX skeleton directory.

Make each STR `format-details` field exactly match the RTCM message types and update periods in the stream. Match its coordinates and generator to the station log. In `misc`, describe provenance: for a pulled stream, include upstream caster, original mountpoint, and pull count; for an active upload, identify the submitting organization.

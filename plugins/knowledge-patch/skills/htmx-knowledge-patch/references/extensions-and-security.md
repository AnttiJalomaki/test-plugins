# Extensions and Security

## CSP nonce for generated styles

`inlineStyleNonce` applies a Content Security Policy nonce to inline styles
generated by htmx. Configure it before htmx loads, for example with the
configuration meta tag:

```html
<meta name="htmx-config" content='{"inlineStyleNonce":"request-specific-nonce"}'>
```

The nonce value must match the policy for the current response.

## Arbitrary attribute mutation

The `attribute-tools` extension swaps arbitrary attributes on or off through
`attributes` or `data-attributes`. It parallels `class-tools`, which performs
the same kind of transition for CSS classes through `classes` or
`data-classes`.

## Path variables and dynamic URLs

The `path-params` and `dynamic-url` extensions solve different URL-building
problems:

| Extension | Behavior |
| --- | --- |
| `path-params` | Fills path variables from request parameters, then removes consumed parameters from the query string or request body |
| `dynamic-url` | Resolves `{varName}` placeholders through a configurable function or a `window` fallback, without depending on `hx-vals` |

Choose based on where values originate and whether consumed request parameters
must be removed.

## Selective parameter replacement

The `replace-params` extension changes only the named parameters and preserves
all others. An empty-string replacement deletes that parameter. This supports
pagination or search changes without rebuilding the complete parameter set.

## Typed and structured JSON forms

`form-json` preserves numeric and boolean types, Base64-encodes files, and
creates nested JSON from dot or bracket notation in form-control names.

Use `json-enc-custom` for more complex objects, lists, and indexed structures
encoded through control names. Select the encoder according to the required
shape; the custom encoder is not merely an alias for the typed form encoder.

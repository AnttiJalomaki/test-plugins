# HTTP Routing And Middleware

Use this reference for HTTP router matching, middleware behavior, request
handling, and proxy compatibility.

## Router composition and matching

- HTTP routers can form a `parentRefs` hierarchy (since 3.6.0). A root router
  attaches to entry points without selecting a service, an intermediate router
  can have children, and a leaf selects the service. A request reaches a child
  only through its parent, so parent middleware or TLS runs first and can enrich
  the request before the child rule is evaluated.
- `Host` and `HostSNI` accept wildcard names such as `*.example.com`, and routing
  precedence is configurable when routes emitted by different providers compete
  (since 3.7.0).
- `defaultRuleSyntax` and `ruleSyntax` are deprecated (since 3.4.0). Do not build
  new configuration around them.

## ForwardAuth and authentication

- `forwardAuth.LogUserHeader` can expose the authenticated user to access
  logging (since 3.2.0).
- ForwardAuth can preserve the authorization server's `Location` response and
  send the incoming request body to that server (since 3.3.0).
- ForwardAuth can preserve the original request method for the authorization
  request (since 3.4.0).
- `authSignInURL` adds sign-in redirects, and `maxResponseBodySize` bounds the
  authorization response. `ForwardAuth.TrustForwardHeader` is deprecated
  (since 3.7.0).
- Authentication middleware warns when `maxBodySize` is unset (since 3.6.0).
  Configure the limit explicitly when forwarding request bodies.
- ForwardAuth propagates the correct `X-Forwarded-Port` to the authorization
  service as of the 3.6.21 patch batch.
- Authentication middleware drops untrusted underscore-bearing `X-*` headers
  (since 3.7.0).

## Headers, compression, and client IPs

- The Headers middleware can emit `Content-Security-Policy-Report-Only` without
  enforcing the policy (since 3.1.0).
- Compress can negotiate Zstandard when the client advertises `zstd` (since
  3.1.0). Its `encodings` option restricts the set of response encodings that
  may be negotiated (since 3.2.0).
- `ipStrategy` accepts an IPv6 subnet setting so IP-based middleware can
  normalize and evaluate IPv6 clients by subnet (since 3.2.0).
- A global forwarded-header option can disable appending to `X-Forwarded-For`,
  and the server can remove incoming header names containing underscores (since
  3.7.0).

## Error, rate-limit, and retry middleware

- Errors middleware can rewrite the HTTP status while serving an error page
  (since 3.4.0).
- RateLimit can store shared state in Redis to enforce a limit across Traefik
  replicas (since 3.4.0).
- Retry can select response status codes, apply a per-attempt timeout, and opt in
  to retrying non-idempotent methods (since 3.7.0).
- Errors middleware `errorRequestHeaders` selects which original request headers
  are sent to the error service (since 3.7.0).

## Encoded paths and proxy request behavior

- `encodedCharacters` applies a route-level policy for encoded characters. The
  related entry-point options are opt-in, and rejected requests are written to
  access logs (since 3.7.0).
- Prefix stripping uses encoded prefix length and sanitizes the resulting URL.
  The 3.7.7 patch also sanitizes paths generated by `ReplacePathRegex` (tracked
  in the 3.7.0 batch).
- The maximum incoming request-header size is configurable (since 3.2.0).
- HTTP/2 server HPACK table sizes are configurable (since 3.6.0).
- HTTP services can preserve the configured backend server path while proxying
  (since 3.2.0).

## Redirect, CORS, WebSocket, and header fixes

- A 3.3.0 WebSocket-upgrade regression requires disabling HTTP/2 extended
  CONNECT where that release is still deployed:

  ```sh
  GODEBUG=http2xconnect=0 traefik
  ```

- Patched 3.7 behavior lets Gateway API header modifiers change `Host`.
  Redirects preserve the incoming scheme when none is configured and emit the
  configured status. CORS no longer emits a default zero max-age or combines a
  credentialed request with wildcard origin. WebSocket upgrades work with
  `h2c` backends (3.7.0 batch).

# Variants and States

Use this reference when styling data states, ancestors, pointer capabilities, disclosure content, validation, or composed conditions in authored CSS.

## Boolean Data Attributes

The foundational state variants in this section are recorded in source batch `4.0.0`.

A present boolean data attribute can be targeted without an arbitrary-variant expression:

```html
<div
  data-current
  class="opacity-75 data-current:opacity-100"
></div>
```

Use `data-current:` for presence. Keep an explicit value selector when the application distinguishes values rather than simple presence.

## Negate States and Conditions

The `not-*` variant negates ordinary states as well as media and `@supports` conditions.

```html
<div class="not-hover:opacity-75"></div>
<div class="not-supports-hanging-punctuation:px-4"></div>
```

The first rule applies while the element is not hovered. The second applies when the named feature query is not supported.

## Respond to an Implicit Ancestor

`in-*` responds to a matching ancestor in the same spirit as `group-*`, but the ancestor does not need a `group` class.

```html
<div tabindex="0">
  <span class="opacity-50 in-focus:opacity-100">
    Visible on ancestor focus
  </span>
</div>
```

Use `group-*` when a named or explicit grouping boundary is valuable. Use `in-*` when any matching ancestor should activate the state.

## Style an Open Popover

The `open:` variant also targets `:popover-open`, so it styles a native popover while that popover is displayed.

```html
<div popover class="opacity-0 open:opacity-100">
  Popover content
</div>
```

This uses the same `open:` form used for other open-state elements; no separate popover variant is needed.

## Pointing-Device Capabilities

The additional environment and interaction variants below are recorded in source batch `4.1.0`.

| Variant | Match |
|---|---|
| `pointer-fine:` | The primary pointing device is precise, such as a mouse. |
| `pointer-coarse:` | The primary pointing device is coarse, such as touch. |
| `any-pointer-fine:` | At least one connected pointing device is precise. |
| `any-pointer-coarse:` | At least one connected pointing device is coarse. |

```html
<button class="p-2 pointer-coarse:p-4">Select</button>
```

Choose `pointer-*` when the primary interaction should drive the design. Choose `any-pointer-*` when an attached secondary device should also enable the behavior, such as touch on a laptop whose primary pointer is a mouse.

## Target the Details Content Container

`details-content:` styles the content container generated by a `<details>` element. It does not merely select each child placed after the summary.

```html
<details class="details-content:mt-3 details-content:bg-gray-50">
  <summary>Question</summary>
  <div>Answer</div>
</details>
```

This is useful when spacing, backgrounds, or transitions belong to the disclosure's content box as a whole.

## Adapt to Inverted Colors

`inverted-colors:` applies styles when the operating system uses an inverted color scheme.

```html
<div class="shadow-xl inverted-colors:shadow-none">...</div>
```

Use it to disable or replace effects such as shadows that invert poorly rather than trying to infer the mode from a color-scheme preference.

## Apply Styles Without JavaScript

`noscript:` applies CSS when JavaScript is disabled and does not require a `<noscript>` wrapper.

```html
<div class="hidden noscript:block">
  Please enable JavaScript.
</div>
```

The variant is suited to progressive-enhancement fallbacks that should remain in normal document structure.

## Validate After Interaction

`user-valid:` and `user-invalid:` represent validation state after the user has interacted with a control.

```html
<input
  type="email"
  required
  class="border user-valid:border-green-500 user-invalid:border-red-500"
/>
```

These variants avoid the initial error styling that `invalid:` can apply to an untouched required field.

## Stack or Combine Variants in CSS

CSS variant composition is part of the 4.2 feature set represented by source batch `4.3.0`.

A colon stacks variants, requiring all conditions in sequence. A comma applies one declaration block to multiple alternative variants.

```css
.button {
  @variant hover:focus {
    background: var(--color-sky-600);
  }

  @variant active, disabled {
    opacity: 50%;
  }
}
```

In this example the background requires both hover and focus, while the opacity applies for either active or disabled state.
